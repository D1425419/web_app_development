# 讀書筆記本系統 PRD (產品需求文件)

## 1. 專案概述
- **背景與動機**：現代人生活忙碌，雖然有閱讀習慣，但往往讀完就忘，缺乏一個系統化整理閱讀紀錄與心得的工具。因此，我們希望開發一個簡單易用的讀書筆記本系統，幫助使用者沉澱閱讀收穫。
- **目標用戶**：有閱讀習慣的人。
- **核心價值主張**：提供一個專屬的數位空間，讓使用者能輕鬆記錄、回顧與評價每一本讀過的書。

## 2. 功能需求

### 主要功能清單
1. 紀錄書名與基本資訊
2. 上傳書本封面
3. 紀錄閱讀時間
4. 紀錄閱讀心得
5. 書籍評分系統

### 使用者故事 (User Stories)
- **紀錄書名**：作為一位有閱讀習慣的人，我希望能夠輸入並儲存書名，以便未來查找我讀過哪些書。
- **上傳書本封面**：作為一位有閱讀習慣的人，我希望能夠上傳書本的封面圖片，以便在瀏覽書單時有更直覺、美觀的視覺體驗。
- **紀錄閱讀時間**：作為一位有閱讀習慣的人，我希望能夠記錄開始閱讀與完讀的日期，以便了解自己的閱讀速度與習慣。
- **紀錄閱讀心得**：作為一位有閱讀習慣的人，我希望能夠記錄下每本書的閱讀心得與筆記，以便日後回顧與反思。
- **評分**：作為一位有閱讀習慣的人，我希望能夠為讀完的書打分數（如 1-5 顆星），以便快速篩選出我最喜歡的書籍。

## 3. 非功能需求
- **技術限制**：
  - 後端框架：Flask
  - 前端模板：Jinja2
  - 資料庫：SQLite
- **效能與安全考量**：
  - 圖片上傳：需限制上傳檔案大小（例如：不超過 2MB），並驗證檔案格式（僅限 JPG, PNG, GIF），以防惡意檔案上傳或伺服器空間耗盡。
  - 安全性：表單送出需防範基礎的 SQL Injection 與 XSS 攻擊。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須有)**：
  - 紀錄書名
  - 紀錄閱讀心得
  - 書籍評分
- **Should Have (應該有)**：
  - 紀錄閱讀時間（開始與結束日期）
  - 書單列表瀏覽
- **Nice to Have (可以有)**：
  - 上傳書本封面圖片
  - 書籍搜尋與過濾
